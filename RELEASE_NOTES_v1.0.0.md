# Coding Tentacle v1.0.0 — Release Notes

## What is Coding Tentacle?

Coding Tentacle is a **safety-first guardian layer for LLM-based code fixing**. It does NOT autonomously repair code. Instead, it **orchestrates, protects, verifies, explains, and learns** from every fix attempt — whether generated by templates, local LLMs, or cloud agents.

## What CT can do (v1.0.0)

- **Classify bugs** by operational mode: EXCEPTION, SECURITY, ALGORITHMIC, REPO, UNKNOWN
- **Classify GitHub issues**: BUG, FEATURE_REQUEST, ENHANCEMENT, DOCUMENTATION, PERFORMANCE, SECURITY, SUPPORT, MISSING_CONTEXT, NEEDS_REPRODUCTION
- **5-Layer Safety VETO**: ReflexLayer → AST → Semgrep → Bandit → SafetyBrain
- **Root Cause Analysis**: 18 cause classes with historical memory
- **Multi-candidate Tournament**: Compare repair strategies, score by safety+test success
- **LLM Agent Interface**: Local (Ollama) + Cloud stubs with BudgetGuard and PatchOutputParser
- **Learning**: Bayesian trust updates, BLM (SQLite+FTS5), ConsolidationCycle, DeuteroLearning
- **EvidenceLedger**: Immutable audit trail for every decision
- **Context Acquisition**: For unclear issues, determine exactly WHAT information is missing

## What CT cannot do (yet)

- **Auto-apply patches**: HumanApprovalGate REQUIRED — design invariant, never overridden
- **Fix algorithmic bugs semantically**: Needs LLM for code understanding (interface exists)
- **Replace a human developer**: CT is a GUARDIAN, not a coder
- **Run cloud LLMs without API keys**: Cloud adapters are stubs
- **SWE-bench Verified**: Harness exists, real runs pending

## Safety Promise

```

✅ NEVER auto-apply

✅ EVERY decision is auditable via EvidenceLedger

```

## Quick Start

```bash
# Clone
git clone https://github.com/nessos666/coding-tentacle.git
cd coding-tentacle

# Install
pip install -e .

# Test
pytest -q

# Docker
docker build -t coding-tentacle .
docker run --rm coding-tentacle
```

## Architecture

```
Bug Report
    ↓
BugModeRouter → IssueUnderstandingBrain → ContextAcquisitionBrain
    ↓
EXCEPTION → Classifier → RootCause → Patch/LLM
SECURITY  → AST + Semgrep + Bandit → VETO
ALGO      → TestAnalyzer → LLM Tournament
REPO      → SpatialMem + External Agent
    ↓
SafetyBrain (VETO) → EvidenceLedger → HumanApprovalGate
```
